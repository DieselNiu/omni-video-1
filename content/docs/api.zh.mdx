GPT Image 2 API 通过两个接口对外暴露生图能力：`submit` 用于提交任务，`GET /images/{task_id}` 用于轮询结果。每张生成的图片扣 1 积分，与 Web 控制台共用同一积分钱包。

## 鉴权

所有请求必须携带 Bearer Token（在 [API 控制台](/settings/api) 创建）。

```
Authorization: Bearer sk-gi2-<32 位十六进制>
```

创建 Key 的前提是**至少完成过一次积分包购买**。赠送积分、签到积分不解锁 API 访问。

## Base URL

```
https://<your-domain>/api/v1
```

## POST `/images/submit`

提交一个新的生图任务。

### 请求体

| 字段               | 类型     | 必填 | 说明                                                                                           |
| ------------------ | -------- | ---- | ---------------------------------------------------------------------------------------------- |
| `prompt`           | string   | 是   | 最长 4000 字符。                                                                               |
| `n`                | integer  | 否   | 必须为 `1` —— 上游目前只支持单张。每张 1 积分。                                                |
| `size`             | string   | 否   | 取值：`1:1`、`3:2`、`2:3`、`4:3`、`3:4`、`5:4`、`4:5`、`16:9`、`9:16`、`2:1`、`1:2`、`21:9`、`9:21`。别名：`aspect_ratio`。 |
| `resolution`       | string   | 否   | `1k`、`2k` 或 `4k`。`4k` 仅支持 `16:9`、`9:16`、`2:1`、`1:2`、`21:9`、`9:21` 六种比例。         |
| `output_format`    | string   | 否   | `"png"`、`"jpg"` 或 `"jpeg"`。                                                                 |
| `reference_images` | string[] | 否   | 最多 16 张参考图 URL（或 base64 data URI）。别名：`image_urls`。                               |
| `model`            | string   | 否   | 传值时必须为 `"gpt-image-2"`。                                                                 |

### 示例

```bash
curl -X POST https://<your-domain>/api/v1/images/submit \
  -H "Authorization: Bearer $GPTIMAGE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "prompt": "a red fox in a snowy forest, cinematic lighting",
    "size": "16:9",
    "resolution": "2k"
  }'
```

### 响应

```json
{
  "task_id": "a1b2c3d4-...",
  "status": "processing",
  "credits_used": 1
}
```

`task_id` 是内部 UUID，拿去查询接口轮询即可。积分在任务被接受时原子扣除，若 Provider 拒单则自动退款。

## GET `/images/{task_id}`

查询任务状态，每 1–2 秒轮询一次，直到 `status` 为 `completed` 或 `failed`。

### 示例

```bash
curl https://<your-domain>/api/v1/images/a1b2c3d4-... \
  -H "Authorization: Bearer $GPTIMAGE_API_KEY"
```

### 响应

```json
{
  "task_id": "a1b2c3d4-...",
  "status": "completed",
  "images": ["https://cdn.<your-domain>/generated/images/..."],
  "error_message": null,
  "created_at": "2026-01-01T00:00:00Z",
  "updated_at": "2026-01-01T00:00:30Z"
}
```

`status` 取值：`processing`、`completed`、`failed`。返回的图片 URL 始终来自我们自己的 CDN，绝不暴露上游 Provider。

## 错误码

| HTTP | `error`                | 含义                                               |
| ---- | ---------------------- | -------------------------------------------------- |
| 400  | `INVALID_INPUT`        | 参数缺失或非法（prompt、n、reference_images）。    |
| 401  | `UNAUTHORIZED`         | Bearer Key 缺失、无效或已撤销。                    |
| 402  | `INSUFFICIENT_CREDITS` | 积分不足。                                         |
| 404  | `NOT_FOUND`            | 任务不存在或不属于当前调用者。                     |
| 502  | `PROVIDER_ERROR`       | 上游 Provider 错误，可退款时会自动退回积分。       |

## 定价

一张图 1 积分，`n=2` 扣 2 积分。积分包每张图最低 $0.05（Enterprise 3,300 包），入门 $0.072（Standard 500 包）。积分永不过期，API 与 Web 控制台共用余额。
